RPDB Developer Documentation

In order to use the RPDB API you must first subscribe to one of the Tiers on Patreon.
An example of the images that are made available trough this API is available on the Examples Page.
The request limit associated with each tier is counted for each image request, upon passing the request limit the service will respond with an error. Caching of images is allowed for personal use and recommended for larger projects.
Images can be requested by either IMDB ID, TMDB ID or TVDB ID.

Within these Docs

· {api-key} should be replaced with your RPDB API Key;

· {id-type} should be replaced with either: "imdb", "tmdb" or "tvdb";

· {media-id} should be replaced with the string ID of the media in question depending on the chosen {id-type};

· all requests should be made to the following host: https://api.ratingposterdb.com/;

· adding ?fallback=true to any of the requests tells the servers to always respond with a poster, even when ratings are not yet available;

API Keys

· easily distinguish between the tiers of API Keys by the first 3 characters of the API Key: t1- (tier 1), t2- (tier 2), etc;

· verify the validity of an API Key by making a GET request to the URL path: /{api-key}/isValid, this path will respond with a 403 HTTP error if the API Key is invalid;

Tier 1 - URL Paths

· /{api-key}/{id-type}/poster-default/{media-id}.jpg

Tier 2 - URL Paths

· /{api-key}/{id-type}/poster-certs/{media-id}.jpg
· /{api-key}/{id-type}/poster-rt/{media-id}.jpg
· /{api-key}/{id-type}/poster-mc/{media-id}.jpg

Tier 3 - URL Paths

· /{api-key}/{id-type}/backdrop-default/{media-id}.jpg
